# Cloudflare R2 implementation guide (Phase 1)

This guide sets a concrete image storage choice and conventions for **catch** and **fishing site** workflows.

## 1) Storage choice

- **Object storage**: Cloudflare R2 bucket (private).
- **Upload method**: short-lived signed upload URLs generated by backend.
- **Read method**: signed read URLs for private media, or Cloudflare Images/public custom domain if you decide media can be public.

## 2) Key conventions

Use deterministic keys to keep storage organized and auditable.

- Catch image key: `users/{userId}/catches/{catchId-or-temp}/{timestamp}-{slug}.jpg`
- Site image key: `users/{userId}/sites/{siteId-or-temp}/{timestamp}-{slug}.jpg`

Persist both:
- `storageKey` (canonical object key), and
- `url` (resolved read URL, usually signed and short lived).

## 3) Backend integration steps

1. Create R2 bucket (private).
2. Create API token with R2 write/read limited to that bucket.
3. Add env vars in server:
   - `CLOUDFLARE_ACCOUNT_ID`
   - `CLOUDFLARE_R2_ACCESS_KEY_ID`
   - `CLOUDFLARE_R2_SECRET_ACCESS_KEY`
   - `CLOUDFLARE_R2_BUCKET`
   - `CLOUDFLARE_R2_PUBLIC_BASE_URL` (optional if using custom public domain)
4. Add endpoints:
   - `POST /api/uploads/sign` (returns signed PUT URL + storage key)
   - `POST /api/uploads/complete` (optional finalization/metadata hook)
5. Client flow:
   - request signed URL,
   - upload directly to R2,
   - submit `storageKey` + final URL with catch/site create payload.

## 4) DB conventions in this repo

- Images are represented by `Image` rows and linked through `CatchImage` / `SiteImage` join tables.
- Catch/site create APIs already accept `images[]` with `storageKey` + `url` and persist them.

## 5) Security and lifecycle

- Signed URL expiry: 1–5 minutes for uploads, 5–15 minutes for reads.
- Limit upload MIME types (`image/jpeg`, `image/png`, `image/webp`) and max size.
- Keep bucket private by default.
- Add a cleanup job for orphaned images (uploaded but never linked).

## 6) Phase 1 done checklist

- [ ] Upload from Log Catch works and image appears on catch detail.
- [ ] Upload from Log Site works and image appears on site detail.
- [ ] `storageKey` saved for each image.
- [ ] Private bucket, signed URL expiry validated.
